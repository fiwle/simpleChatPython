# Simple Chat Python with Asyncio

Простой асинхронный чат с одним сервером и несколькими клиентами. Реализован на Python 3.10+ без использования сторонних библиотек.

## Возможности

- Обмен сообщениями между всеми подключенными клиентами
- Приватные сообщения через `@[username]`
- Передача файлов с присваиванием каждому файлу уникального ID (файлы хранятся в оперативной памяти, при остановке сервера загруженные файлы удаляются)
- Уведомления при подключении и отключении клиентов

## Требования
- Python 3.10+

## Структура
pythonChat/
- server.py
- client.py
- downloads/ # Создается автоматически
- README.md

## Запуск

### 1. Сервер

```bash: python server.py [адрес] [порт]```

Параметры по умолчанию: ```127.0.0.1``` и ```8888```.

### 2. Клиент

Параметры могут передаваться аргументами в командной строке или интерактивно.

С аргументами:

```python client.py 127.0.0.1 8888 misha```

Без аргументов:
```
python client.py
Server address [127.0.0.1]:
Port [8888]:
Your username: misha
```

## Использование

После подключения пользователя выводится уведомление о его подключении на сервере и остальным клиентам.

После этого пользователь может отправлять сообщения.
Для отправки сообщения необходимо с терминала клиента напечатать текст и отправить его через Enter.
После отправки сообщение увидят все клиенты, подключенные к комнате, в формате:

```[Username]: Message```

### Приватные сообщения

Для отправки приватного сообщения, используйте символ `@` перед именем пользователя:

```@misha Привет, это только для тебя```

Или же можно использовать команду:

```/private misha Привет, это только для тебя```

При этом получателю и отправителю сообщение придет с пометкой о приватности.

Если пользователь не существует, то сервер сообщит от ошибке.

### Загрузка файлов

Для отправки файлов используется команда `/file [путь]`. Например, на Windows:

```/file C:\Users\misha\Photos\Anapa2007.jpeg```

После отправки сервер сохранит файл в оперативной памяти, присвоит ему уникальный ID и вышлет остальным участникам уведомление.

### Скачивание файлов

Файл, загруженный клиентом, можно скачать с помощью команды `/download [ID]`.

После загрузки файл будет сохранен в папку `downloads/` с исходным именем. Если файл с таким именем уже существовал, то к нему добавится номер (например, `Anapa2007_1.jpeg`).

### Дополнительные команды

- `/help` -- показать список команд
- `/quit` -- выйти из чата (также с помощью *Ctrl+C*).

Для выхода из клиента можно либо отправить команду ```\quit```, либо нажать *Ctrl+C*. При отключении пользователя повляется уведомление
```User [name] left the chat.```

Для остановки сервера *Ctrl+C* в терминале сервера.

## Ограничения

- Файлы хранятся в оперативной памяти. После перезагрузки все файлы исчезают. Возможно реализовать работу с БД.
- Передача файлов между всеми клиентами, нет личного обмена между клиентами.
- Нельзя изменять имя файла, оно остается таким же
- Приватные сообщения не шифруются 

## Особенности реализации

- Асинхронность -- сервер и клиент используют ```asyncio``` для работы с множеством клиентов.
- Декомпозиция -- сервер разбит на ```ChatServer``` и ```ClientSession```, клиент использует отдельные задачи для вывода и ввода.
- Надежность -- обработаны возможные обрывы соединения, дубликаты имен и некорректный ввод, для передачи файлов используется `readexactly()` для гарантии целостности данных.
